Security + JWT + JPA(H2)

Java 프레임워크인 Spring Framework를 이용하여 Security + JPA 로 JWT를 이용한 회원 / 게시판 CRUD 적용

프로젝트를 통해 배운 내용

1. Spring Framework 이해
- 기존에 수행했던 Express의 model, controller, route의 형태를 유지하면서 객체지향이 이루어지도록 작성함

2. 기타
- 의문사항 :
a. security에 자주 보이는 role은 무슨 역할인가?
b. permitAll()을 적용해야지만 api 통신이 되는건가?
c. SecurityContextHolder 에 사용자 정보를 등록 할 경우 무상태성(Stateless)이 유지 되는가?

종합 : Spring Framework 이용하여 REST API를 만드는 기술을 습득하면서 많은 의문이 생겼다.
