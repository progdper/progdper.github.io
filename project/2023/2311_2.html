careme_dual Website

Express + Apollo Server 를 기반인 Graphql 구조이며 TypeGoose + MongoDB를 사용하여

생성된 관리자가 게시판 CRUD를 하고 auto-increment 라이브러리를 이용하여 _id를 number 처리함.

프로젝트를 통해 배운 내용

1. 데이터베이스 설계
- MongoDB 구조를 작성 할 때에는 관계형처럼 형식에 얼매이지 않아도 되며 Read를 빠르게 할 수 있다.
- 순차적으로 데이터를 등록하지 않기에 auto-increment 라이브러리를 사용해야 원하는 형태가 나온다. 그리고 auto-increment 는 DB에 등록하여 사용함에 DB를 빠르게 Read 할 수 있는 장점을 이용하여 관계형 DB의 형태를 만들 수 있다.
- 에피소드 :
a. local에 등록하는 MongoDB를 사용하고자 함에 Prisma ORM를 이용할 경우 Replica Set 문제가 발생하여 도커를 이용해야 하는 방법만 된다.(MongoDB Atlas 지원)
b. TypeGoose를 이용하여 DB Modeling 할 때에 auto-increment 사용하는 방법을 찾는데 많은 시간이 필요했음.

2. GraphQL 및 TypeGraphql
- Apollo Server 를 이용하면서 context에 TypeGoose 를 등록하여 쉽게 호출 할 수 있다.
- TypeGraphql의 middleware를 이용하여 사용제한을 할 수 있다.
- input과 output 에 대하여 협의가 되지 않은 상태에서 개발을 하다보니 REST API 보다 GraphQL이 더 효율적으로 진행 할 수 있게 지원한다고 생각한다.

종합 : 새로운 기술들을 배울 때에는 상대적으로 많은 시간이 필요하지만 더 쉽고 빠르게 구축할 수 있게 된다..
